## Context & Motivation

The architecture (tech-decisions §1) is a **PWA** — but there's no web manifest,
icons, or service worker, so it isn't installable. Add them so "Bra!" can be
installed to the home screen (the spec's distribution path, before any Capacitor wrap).

## Affected Components
- Add: `vite-plugin-pwa` (devdep) OR a hand-rolled manifest + SW
- Create: app icons (192×192 + 512×512 PNG, ideally a maskable variant), `manifest.webmanifest`
- Modify: `vite.config.ts` (PWA plugin config), `index.html` (theme-color, apple-touch-icon if hand-rolled)
- Dependencies: none; Blocking: none

## Approach
- Prefer `vite-plugin-pwa` (`bun add -d vite-plugin-pwa`) with `registerType: 'autoUpdate'` and a `manifest`:
  - `name: "Bra! — Dog Training"`, `short_name: "Bra!"`, `description`, `theme_color` (match the green/sky), `background_color`, `display: "standalone"`, `orientation: "portrait"`, `start_url: "/"`, icons (192/512 + maskable).
- ICONS: create a simple but real icon (e.g. a green rounded square with a paw/"Bra!"). You can RENDER an SVG to PNG using the available headless Chromium via playwright-core (we already use it for screenshots: write an SVG, open it as a data URL, screenshot at 192 and 512) — that produces real PNGs without extra deps. Maskable icon = same art with safe padding. Put icons in `public/`.
- Ensure the SW caches the app shell (the plugin's Workbox default precache is fine).
- Merge config carefully — do NOT clobber the existing `build.rollupOptions.manualChunks` or the Vitest `test` block in vite.config.ts.

## Verification (reuse the running dev server for the app; build for the artifacts; do NOT pkill; never fake a screenshot)
- `bun run build` emits `manifest.webmanifest` + a service worker + the icons into `dist/`.
- `index.html` (built) links the manifest + has theme-color.
- The app still loads + renders (screenshot a training frame). VIEW it.
